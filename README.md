# Mini Daily Planner

A minimalist, local-first daily planner built entirely by AI agents to test a custom OpenCode multi-agent workflow.

**Live Demo:** https://TheDoomGuy-Dev.github.io/mini-daily-planner/

---

## What This Is

A fully functional daily planner with:
- Task CRUD (create, edit, complete, delete)
- Drag-and-drop reordering
- Recurring tasks (daily/weekly/custom)
- Reminders with browser notifications
- Task templates
- Light/dark/system themes
- Import/export JSON
- Bulk actions
- Search & filtering
- Keyboard shortcuts

**All data stays in your browser** — no accounts, no cloud, no tracking.

## How This Was Built

This app was **entirely generated by AI agents** using a custom **OpenCode Professional MVP Workflow v2.7**. No human wrote production code.

### The Workflow (16 Phases)

| Phase | Agent | Output |
|-------|-------|--------|
| 1 | Requirements Interviewer | Product concept, user stories, tech stack decisions |
| 2 | Product Manager | PRD, acceptance criteria (45 criteria) |
| 3 | UX Designer | User flows, screen specs, interactions |
| 4 | Visual Design Analyst | Design system, component specs, color palette |
| 5 | Architect | Architecture, data model, implementation plan, test strategy |
| 6 | Critic | Architecture review, risk register, 13 findings |
| 7 | Builder | Full implementation (15 vertical slices) |
| 8 | Build Verifier | Type-check, lint, tests, build verification |
| 9 | QA Verifier | Manual testing against 45 criteria |
| 10 | Debugger | Fixed 13 bugs across 2 cycles |
| 11 | Build Verifier (re-verify) | Clean build confirmation |
| 12 | QA Verifier (cycle 2) | Regression testing |
| 13 | Debugger (cycle 2) | Fixed 4 remaining bugs |
| 14 | Build Verifier (final) | Clean build + visual fixes |
| 15 | Visual QA | 20 screenshots analyzed, 3 issues fixed |
| 16 | Release Auditor | Final audit → **RELEASE APPROVED** |

### Key Stats

| Metric | Value |
|--------|-------|
| Context tokens used | ~200k |
| Phases completed | 16 |
| Bugs fixed | 13 (6 critical, 4 high, 3 medium) |
| Visual issues fixed | 3 |
| Acceptance criteria | 45/45 passed (100%) |
| Automated tests | 28/28 passing |
| Human code written | **0 lines** |

### The Workflow in Simple Terms

1. **Requirements** → AI interviews you, captures all decisions
2. **Design** → AI creates UX flows, visual system, component specs
3. **Architecture** → AI designs data model, APIs, implementation plan
4. **Critic** → AI independently reviews for risks, bugs, gaps
5. **Build** → AI implements in vertical slices with tests
6. **QA/Debug cycles** → AI tests, finds bugs, fixes, re-tests (max 2 cycles)
7. **Visual QA** → AI compares screenshots to design specs
8. **Release audit** → AI verifies everything, approves release

Each phase has **gate criteria** — the next phase only starts when the current one passes validation. The workflow uses **Scribe** for long document writes and **Omnissiah** as a final escalation advisor.

## Tech Stack

- **Framework:** React 18 + TypeScript + Vite
- **Styling:** Tailwind CSS + shadcn/ui + Radix UI
- **State:** React Context + useReducer
- **Persistence:** localStorage (zero backend)
- **Drag & Drop:** @dnd-kit
- **Dates:** date-fns
- **Testing:** Vitest + React Testing Library
- **Icons:** Lucide React

## Running Locally

```bash
git clone https://github.com/TheDoomGuy-Dev/mini-daily-planner.git
cd mini-daily-planner
npm install
npm run dev
```

Open http://localhost:5173
